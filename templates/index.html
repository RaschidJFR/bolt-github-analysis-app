Repository Issue Analyzer

A tool for analyzing and prioritizing GitHub issues based on traction, impact, and estimated effort. This tool helps project maintainers identify which issues to focus on first.

Analysis Progress

The analysis will generate a CSV report containing the top 15 issues based on the algorithm. For more information, visit the GitHub repository.

How to read the report

The analysis returns a csv with the following columns:

Main Fields

Column Type Description
number numeric GitHub issue number
url text GitHub issue URL
title text Issue title
issue_type text One of: Bug, Security, Performance, Documentation, Feature Request, Other
summary text Concise summary of the issue and its conclusion
score numeric Overall priority score (normalized 0-1, higher = more important). The resulting list is sorted by this value.
Formula: traction * impact / effort
traction numeric Normalized score based on comments, reactions, and engagement. Useful for highlighting the most popular issues.
impact numeric Impact score, from 1 to 4, according to an LLM.
impact_analysis text Explanation of impact score assignment.
effort numeric Effort estimation from LLM analysis.
Note: This estimation can be improved with a dedicated prompt but for now it seems to be enough for a decent issue ranking.
createdAt text ISO timestamp when issue was created

Additional Fields

Column Type Description
avg_comments_per_week numeric Average number of comments per week over recent period
commentCount numeric Total number of comments (excluding repo members)
commenterCount numeric Number of unique commenters (excluding repo members)
reactionCount numeric Total number of reactions on issue and comments
last_comment text ISO timestamp of most recent comment

Note: This tool uses AI to analyze issues, so results may vary. Always review the analysis before making important decisions.
